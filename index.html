Matching Special Characters

There may be cases when you need to match special characters, like metacharacters.

  • \ (backslash)
  • ^ (caret)
  • $ (dollar)
  • . (dot)
  • | (vertical bar)
  • ? (question mark)
  • * (asterisk)
  • + (plus)
  • [ ] (square brackets)
  • {} (curly brackers)

The solution is to escape the metacharacters by preceding them with a backslash .

Character Classes

Character Classes (also called Character Sets) allow matching a range of characters. Character classes are defined using [ and ] .

For example, [XYZ] will match any of the three characters:

One of the most popular use cases of sets is matching both, upper and lowercase letters.

Range of Characters

RegEx supports character ranges, which can be defined using a dash (-) between two values. [a-z] [A-Z]

[A-Za-z0-9] is the same as :

[ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789].

Negated Character Classes

The caret (^) metacharecter can be used to deny a character set

Digits (and non-Digits)

[0-9] and [^0-9]

\d - shorthand for [0-9]

\D -shorthand for [^0-9] (any non-digit), the same as [^\d]
